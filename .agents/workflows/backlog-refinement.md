# Backlog Refinement Workflow

Weekly backlog grooming session to ensure upcoming sprint stories are ready.

## Pre-Session (PO — 1 day before)
1. Review and prioritize backlog items
2. Add acceptance criteria to top 10 stories
3. Attach mockups/designs (if applicable)
4. Mark stories needing team discussion

## Session (1 hour, weekly)

5. **PO presents each story** (3-5 min per story)
   - Business context and value
   - Acceptance criteria walkthrough
   - Open questions

6. **Team asks clarifying questions**
   - "What happens when...?"
   - "Does this include...?"
   - "What about edge case...?"

7. **Estimation (Planning Poker)**
   - Each member shows estimate simultaneously
   - If estimates differ by > 2x, discuss and re-estimate
   - Use Fibonacci scale: 1, 2, 3, 5, 8, 13
   - Stories > 8 points → break down into smaller

8. **Break down large stories**
   - Split by user workflow steps
   - Split by data entity (backend vs frontend)
   - Split by happy path vs edge cases

9. **Identify dependencies**
   - Which stories block other stories?
   - Which need design/API first?
   - Cross-team dependencies?

## Post-Session

10. **PO updates backlog**
    - Stories refined and estimated
    - Dependencies documented
    - Priority confirmed for next sprint

11. **Definition of Ready checklist**
    - [ ] Story has clear description
    - [ ] Acceptance criteria complete
    - [ ] Story estimated (≤ 8 SP)
    - [ ] Dependencies identified
    - [ ] Design/mockups attached (if needed)
    - [ ] Technical approach discussed
