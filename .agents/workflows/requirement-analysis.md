# Requirement Analysis Workflow

Structured process for gathering, analyzing, and documenting requirements led by the Business Analyst.

## Steps

1. **Identify Stakeholders**
   - List all affected users and decision makers
   - Create RACI matrix for the feature
   - Schedule interviews/workshops

2. **Conduct Stakeholder Interviews**
   - Use BA interview template from business-analyst skill
   - Focus on: current process, pain points, desired outcomes
   - Document key findings and quotes

3. **Analyze Current Process**
   - Map existing workflow (As-Is process)
   - Identify bottlenecks and inefficiencies
   - Document workarounds people use today

4. **Define Requirements**
   - Write functional requirements (FR-xxx)
   - Write non-functional requirements (NFR-xxx)
   - Define business rules (BR-xxx)
   - Create data dictionary for new entities

5. **Create User Stories**
   - Follow INVEST criteria from product-owner skill
   - Format: "As a [role] I want [feature] so that [value]"
   - Each story ≤ 8 story points

6. **Write Acceptance Criteria**
   - Use Given-When-Then format
   - Cover happy path and edge cases
   - Include validation rules

7. **Create Process Map (To-Be)**
   - Design the future workflow
   - Highlight automation opportunities
   - Validate with stakeholders

8. **Review & Sign-off**
   - BA presents requirements to PO and Tech Lead
   - PO confirms business value and priority
   - Tech Lead confirms technical feasibility
   - Stakeholders approve requirements document

9. **Handoff to Development**
   - Requirements doc shared with team
   - Tech Lead creates technical tasks
   - QA creates test plan from acceptance criteria
